# Blog

A Go modular monolith (github.com/example/blog) with a React admin frontend, generated by [brrr](https://github.com/amronrails/brrr).

## Stack

- **Backend:** Go · chi · sqlc + pgx · Postgres · goose · JWT auth (golang-jwt v5) · bcrypt
- **Frontend:** Vite · TypeScript · TanStack Query · React Router · shadcn/ui + Tailwind (`web/`)

## Architecture

A modular monolith using ports & adapters. Each feature lives under
`internal/modules/<name>` as a vertical slice with a public `module.go` facade
and a private `internal/` package; dependencies point inward.

```
adapters/http ─┐
               ├─→  services  →  ports  →  domain
adapters/postgres ─┘            (interfaces) (entities,
(handlers,        (sqlc-backed   the services  value objects,
 DTOs / repos)     persistence)   depend on)    errors — pure)
```

The only public surface of a module is `module.go`, which wires the slice and is
mounted via `internal/modules/registry.go`. See [AGENTS.md](AGENTS.md) for the
full layout and conventions.

The built-in **user** module provides email/password auth with short-lived JWT
access tokens and rotating, server-revocable refresh tokens. The first account
to register becomes the **admin**.

## Getting started

```sh
cp .env.example .env       # then edit JWT_SECRET
make db-up                 # start Postgres
make setup                 # go mod tidy + sqlc generate + npm install
make migrate               # create the schema
make dev                   # run backend (:8080) + frontend (:5173)
```

Open http://localhost:5173 and register the first user — they become admin.

## API

| Method | Path                      | Auth        |
| ------ | ------------------------- | ----------- |
| POST   | `/api/v1/auth/register`   | public      |
| POST   | `/api/v1/auth/login`      | public      |
| POST   | `/api/v1/auth/refresh`    | public      |
| POST   | `/api/v1/auth/logout`     | public      |
| GET    | `/api/v1/auth/me`         | bearer      |
| GET    | `/api/v1/admin/users`     | bearer + admin |

## Growing the app

```sh
brrr generate <module> <Model> field:type ...   # scaffold CRUD (backend + frontend)
brrr init <app> --spec spec.yaml                 # scaffold a whole app from a YAML spec
```
